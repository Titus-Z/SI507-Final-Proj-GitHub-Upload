# 09. 如何演示这个期末项目

阅读时间建议：20-30 分钟

## 这份文档的目标

读完这一份，你应该知道：

1. 如果老师当场让你演示，你应该按什么顺序讲
2. 哪些功能最适合展示
3. 哪些内容要讲，哪些不要讲太多
4. 如何用 5-10 分钟把项目讲清楚

## 1. 演示时最重要的原则

不要从代码开始讲。

不要从 API 开始讲。

不要从模型名字开始讲。

老师最先关心的通常是：

- 你在做什么
- 为什么这样做
- 结果是什么

所以正确顺序应该是：

```text
问题 -> 设计 -> 图 -> 功能 -> 结果
```

## 2. 一个最稳的 5 分钟讲法

### 第一步：一句话定义项目

先说：

> 这是一个基于真实股票价格和新闻数据的图探索系统，用来分析股票、行业和新闻主题之间的结构关系，而不是预测股价。

这句话一定要先立住。

### 第二步：解释为什么用图

接着说：

> 因为我关心的不是单个股票，而是股票、行业和新闻主题之间的关系，所以图比普通表格更适合作为主组织结构。

### 第三步：讲三类节点和四类边

这一部分不要展开太细，只说核心：

- 节点：股票、行业、主题
- 边：股票相关性、股票-行业、股票-主题、行业-行业

### 第四步：演示一到两个查询

最推荐的顺序是：

1. 查询一只股票，比如 `AAPL`
2. 比较两只股票，比如 `AAPL` 和 `MSFT`
3. 再展示路径或中心性

### 第五步：展示新闻影响层

打开 `News Impact` 页签，说明：

- 哪些行业受新闻影响最多
- 哪些股票受新闻影响最多
- 这些不是价格预测，而是结构化影响分析

## 3. 最适合演示的入口

### 首选：查询型网页

命令：

```bash
streamlit run query_app.py
```

原因：

- 最直观
- 页面整洁
- 功能集中
- 适合课堂演示

### 第二选择：可拖拽图网页

命令：

```bash
streamlit run interactive_graph_app.py
```

这个适合在后半段补充展示：

- 网络结构
- 节点团块
- 拖拽视觉效果

### 备用：CLI

命令：

```bash
python main.py
```

CLI 更适合作为备用方案，因为它最稳定。

## 4. 你应该优先展示哪些功能

不是所有功能都需要展示。

如果时间有限，优先这 5 个：

1. Graph Summary
2. Stock Search
3. Compare Stocks
4. Top Central Nodes
5. News Impact

这五个已经足够把项目的核心价值讲完整。

如果老师对 LLM 很感兴趣，你再加第 6 个：

6. New News Analysis

## 5. 演示 `News Impact` 时该怎么说

你应该这样表述：

> 我把新闻先做成结构化主题层，再额外用 LLM 做离线影响抽取。网页里展示的是本地保存的影响结果，而不是实时调用模型。行业和股票影响分数来自一个显式规则：强度乘以置信度，再结合方向得到净影响分数。

这样讲会比“我用了 OpenAI API”强得多。

## 6. 演示 `New News Analysis` 时该怎么说

这个页面适合当作加分项来展示，不要一上来就先放它。

比较稳的说法是：

> 除了离线新闻影响层，我还做了一个单条新新闻评估入口。用户输入新的新闻标题和摘要后，系统会让 LLM 在当前图的股票、行业和主题范围内做一次结构化 impact assessment，再把结果映射回已有图节点。这样它仍然是图系统的一部分，而不是一个脱离数据约束的聊天框。

如果老师追问“这是不是预测”，你就直接回答：

> 不是。这里做的是结构化影响评估和图对齐解释，不是未来收益预测。

## 7. 老师可能会问你的问题

### 问题 1：为什么不用文章直接做节点？

答：

因为文章节点太碎、数量太大、噪音也高，第一版主图用 topic 节点更稳定、更容易解释。

### 问题 2：为什么股票之间用相关性？

答：

因为它简单、可解释，而且非常适合作为市场结构的第一版关系定义。

### 问题 3：为什么要把数据下载和分析拆开？

答：

为了稳定、可重复和便于调试，避免运行时受 API 限流和网络波动影响。

### 问题 4：LLM 这一层是不是在预测股价？

答：

不是。它只做新闻语义到行业/股票影响标签的结构化抽取，不做价格预测。

## 8. 演示时不要犯的错误

### 错误 1：一上来讲实现细节

这样会让老师先迷失在文件名里，而不是先理解项目价值。

### 错误 2：把 LLM 说成核心

这会让项目看起来像“调 API”，而不是图系统。

### 错误 3：把影响分析说成预测

这会直接把项目带偏。

### 错误 4：展示太多页面

控制在一两个主要界面就够了。

## 9. 一个比较稳的演示脚本

你可以照着这个顺序：

1. 先开 `query_app.py`
2. 说项目目标
3. 指一下 Graph Summary
4. 查 `AAPL`
5. 比较 `AAPL` 和 `MSFT`
6. 展示 Top Central Nodes
7. 打开 `News Impact`
8. 如果老师对 LLM 很感兴趣，再打开 `New News Analysis`
9. 最后切到拖拽图页面补一个整体结构图

这套流程很稳，不容易乱。

## 10. 你现在应该能讲出来的话

> 如果我要在课堂上演示，我会先用一句话定义项目，再说明为什么用图，然后用查询网页展示股票查询、股票比较、中心节点和新闻影响；如果老师对 LLM 很感兴趣，我会再展示单条新新闻评估页面；最后再用可拖拽图页面补充整体结构。这样能在有限时间内同时展示数据、图结构、分析功能和增强型新闻理解层。

如果你能按这个顺序讲出来，课堂演示就不会乱。
