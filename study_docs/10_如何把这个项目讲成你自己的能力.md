# 10. 如何把这个项目讲成你自己的能力

阅读时间建议：20-30 分钟

## 这份文档的目标

读完这一份，你应该能做到：

1. 把这个项目讲成你自己的项目，而不是“别人帮我搭的东西”
2. 知道简历上应该强调什么
3. 知道答辩时应该强调什么
4. 知道你真正学到了哪些能力

## 1. 先分清“做项目”和“讲项目”不是一回事

很多人把项目做完了，但不会讲。

结果就是：

- 代码有了
- 页面有了
- 结果也有了

但一到简历和面试，只会说：

“我做了一个股票分析网站。”

这句话几乎没有信息量。

你真正应该讲的是：

- 你建模了什么对象
- 你怎么组织数据流
- 你为什么做这些设计取舍
- 你怎么验证结果

## 2. 这个项目到底体现了哪些能力

### 能力 1：数据工程能力

你不是手工写死数据，而是设计了：

- 数据获取
- 本地缓存
- 本地读取

这说明你理解真实项目里“分析前先把数据层搭稳”的重要性。

### 能力 2：图建模能力

你不是只做表格分析，而是把股票、行业和新闻主题组织成多层图。

这说明你理解：

- 节点和边的定义
- 关系建模
- 结构分析

### 能力 3：分析接口设计能力

你不是只输出一个图对象，而是封装出了：

- 股票查询
- 行业查询
- 股票比较
- 路径分析
- 中心性分析

这说明你能把底层结构变成用户可使用的功能。

### 能力 4：交互设计能力

你不是只有后台逻辑，还设计了：

- CLI
- 查询型网页
- 拖拽图网页

这说明你理解不同入口的职责，而不是把所有东西堆在一个页面里。

### 能力 5：现代 AI 工具使用能力

你加了 LLM enrichment，但不是滥用。

它被明确限制为：

- 新闻影响结构化抽取
- 离线结果保存
- 网页中的影响展示

这说明你不是“调个 API 玩玩”，而是把模型合理嵌入系统。

## 3. 简历上应该怎么写

不要写：

- Built a stock website.
- Used APIs to analyze stocks.

这种写法太弱。

更好的写法应该突出：

- 多层图
- 真实数据
- 结构化新闻处理
- 图分析功能
- LLM 影响抽取
- 交互系统

一个更像样的表达可以是：

> Built a multi-layer market graph linking stocks, sectors, and news topics from real financial and news data, with graph-based querying, path analysis, centrality analysis, and offline LLM-powered news impact extraction.

如果要再细一点，可以拆成几条 bullet。

## 4. 面试或答辩时最值得强调的点

### 点 1：为什么图是主框架

因为项目关心的是关系，而不是孤立对象。

### 点 2：为什么新闻先做 topic 层

因为 article 层太碎，topic 层更稳定、更适合作为第一版主图新闻层。

### 点 3：为什么数据获取和分析拆开

为了稳定、可重复和便于调试。

### 点 4：为什么 LLM 是 enrichment，不是 prediction

因为这一步的任务是影响结构化，而不是收益预测。

### 点 5：为什么做测试

因为你不是在写演示脚本，而是在做一个多层系统。

## 5. 你真正应该说“我做了”的是什么

当你讲项目时，不要只说功能。

你应该说你做了这些判断：

- 我决定用图作为主组织框架
- 我决定用 topic 而不是 article 作为主新闻层
- 我决定把数据获取和分析拆开
- 我决定用相关性构建第一版股票关系
- 我决定把 LLM 做成离线 enrichment
- 我决定把查询网页和拖拽图网页分开

真正的能力不只是“写了代码”，而是“做了合理的系统设计选择”。

## 6. 你现在最该避免的表述

### 错误表述 1

“这个项目是用来预测股票的。”

错。

### 错误表述 2

“我主要做的是网页。”

也不对。

网页只是入口，不是核心。

### 错误表述 3

“我就是调了一个 OpenAI API 来分析新闻。”

这会让你的项目显得很浅。

你应该强调的是：

- schema
- offline enrichment
- local caching
- structured impact display

## 7. 你真正学到的东西

如果你认真做完并理解这个项目，你学到的其实不是一个功能点，而是一整套方法：

- 如何把真实数据组织成系统
- 如何把新闻变成结构化表
- 如何设计图里的节点和边
- 如何在图上做分析接口
- 如何把后台逻辑接成交互层
- 如何把 LLM 作为增强层而不是噱头
- 如何用测试提升可信度

这套东西比单个 API 或单个页面更有价值。

## 8. 你现在应该能讲出来的话

> 这个项目展示了我在数据建模、图结构设计、系统分层、交互入口设计以及 LLM 结构化应用方面的能力。它不是一个简单的网页项目，也不是一个预测项目，而是一个以图为核心、基于真实金融和新闻数据构建的关系探索系统。我不仅实现了核心数据流和分析功能，还通过本地缓存、测试和离线 enrichment 提升了系统的稳定性和可解释性。

如果这段话你能自然讲出来，那么这个项目就已经能开始为简历服务了。
