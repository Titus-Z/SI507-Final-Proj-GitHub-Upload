# 07. LLM 新闻影响分析层

阅读时间建议：20-30 分钟

## 这份文档的目标

读完这一份，你要理解：

1. LLM 在这个项目里到底负责什么
2. 为什么它是 enrichment，而不是预测器
3. 它输出了哪些结构化结果
4. 网页里“影响多大”是怎么计算和展示的
5. 新输入的一条新闻如何被评估到当前图结构里

## 1. LLM 在这个项目里不是核心骨架

这个判断一定要先明确。

项目的核心骨架仍然是：

- 数据
- 新闻处理
- 图
- 图分析

LLM 是加在这个骨架上的增强层。

也就是说，它不是拿来替代图，也不是拿来做股价预测，而是拿来做：

**新闻语义到结构化影响标签的转换。**

## 2. 为什么要加 LLM

原始新闻里虽然已经有：

- topics
- ticker_sentiment

但它还没有直接告诉你：

- 这篇新闻主要影响哪个行业
- 它对这个行业是正面还是负面
- 这篇新闻更可能影响哪些股票
- 影响强度大概如何

这些信息靠规则可以做一部分，但很快会变得僵硬。

LLM 的价值就在这里：

它可以结合标题、摘要、topic、ticker context，输出更接近人类分析方式的结构化结果。

## 3. 为什么这一步必须做成离线 enrichment

这个设计非常关键。

LLM 没有直接塞进主分析入口，而是单独做成：

- `llm_enricher.py`
- `enrich_news_with_llm.py`

原因很现实：

### 原因 1：成本

调 LLM 是有成本的，不适合每次打开网页都重新跑。

### 原因 2：延迟

如果每次查询都要实时调模型，体验会很差。

### 原因 3：可验证

离线跑完以后保存成 CSV，你可以人工抽查和复用。

所以正确流程是：

```text
本地新闻
-> NewsProcessor
-> LLM enrichment
-> 本地 CSV
-> 查询网站展示
```

除此之外，这个项目现在还多了一条“单条新新闻评估”支线：

```text
用户输入 title + summary
-> LLMNewsImpactAnalyzer
-> 结构化 topic / sector / stock impact
-> 过滤到当前图允许的 universe
-> 查询网站展示
```

## 4. 它输出了什么

这一层最终产出三张表：

### `article_llm_summary`

这张表存文章级总结果，比如：

- 事件摘要
- 主事件类型
- 影响范围
- 市场相关性

### `article_sector_impacts`

这张表存每篇文章对行业的影响：

- `sector`
- `impact_direction`
- `impact_strength`
- `confidence`
- `rationale`

### `article_stock_impacts`

这张表存每篇文章对股票的影响：

- `ticker`
- `impact_direction`
- `impact_strength`
- `confidence`
- `rationale`

## 5. 为什么这里用结构化输出

因为你不是想得到一段“看起来很聪明”的自然语言评论，而是想要后续可以计算、过滤、展示的数据。

所以这层的关键不是 prose，而是 schema。

这也是为什么项目里用了固定枚举：

- `positive`
- `negative`
- `mixed`
- `uncertain`

以及：

- `low`
- `medium`
- `high`

只有这样，后面网站里才有办法做汇总和打分。

## 6. 为什么这不是股价预测

这是答辩时一定要强调的点。

这里的 LLM 输出应该被理解成：

- 新闻语义上的影响判断
- 面向行业和股票的结构化标签

它不是：

- 未来收益预测
- 投资建议
- 买卖信号

你项目的表述应该始终保持在“impact interpretation”而不是“price forecasting”。

## 7. 网页里的“影响多大”到底怎么算

现在查询网站里已经加了 `News Impact` 页签。

这个页面没有直接把 LLM 文本原样展示，而是进一步做了可解释汇总。

核心规则是：

### 第一步：把强度映射成数值

- `low = 1`
- `medium = 2`
- `high = 3`

### 第二步：乘上置信度

得到：

```text
impact_magnitude = strength_weight * confidence
```

这表示影响量级，但还没有方向。

### 第三步：加入方向符号

- `positive = +1`
- `negative = -1`
- `mixed = 0`
- `uncertain = 0`

得到：

```text
net_impact_score = direction_sign * impact_magnitude
```

所以页面会同时显示：

- `gross_impact_score`
- `net_impact_score`

这两个分数一起看才有意义。

## 8. 新的 `New News Analysis` 功能在做什么

除了离线 enrichment 结果，这个项目现在还支持直接输入一条新的新闻标题和摘要。

这个功能的目标不是预测未来价格，而是回答：

- 这条新闻在当前图里最像哪个主题
- 它最可能影响哪些行业
- 它最可能影响哪些股票
- 这些判断的方向、强度和置信度是什么

实现上它有两个约束：

### 约束 1：只在当前图的 universe 里判断

也就是说，LLM 不能随便输出一个系统里不存在的行业、股票、主题。

它只能在：

- 当前股票节点
- 当前行业节点
- 当前主题节点

这些集合里做选择。

### 约束 2：输出后还要再做 graph grounding

也就是说，页面不会只展示模型原话，还会进一步把结果映射到当前图里的：

- 主题节点
- 行业节点
- 股票节点

这样老师一看就能明白：这不是单独漂浮在外面的 LLM 功能，而是图系统里的一个扩展入口。

现在这个功能还多了一步：

### 约束 3：结果会被转换成合成 `topic_stock` 行，再叠加到图里

这一步非常关键。

系统不会把 LLM 原样文本直接塞进图，而是先把它转成和历史新闻层兼容的结构：

- `ticker`
- `topic`
- `article_count`
- `avg_topic_relevance`
- `avg_ticker_relevance`
- `avg_ticker_sentiment`
- `avg_overall_sentiment`

然后再加上一个预览用的显式 `impact_score`。

这样做的结果是：

- 你可以看这条新新闻会新增哪些 `stock-topic` 边
- 也可以看它会强化哪些已有边
- 而且不同新闻、不同股票、不同主题不会被视为“同样重要”

## 9. 新新闻的强弱是怎么区分的

这个项目现在不是把每条新新闻都同等对待，而是用了一个显式公式：

```text
impact_score
= topic_confidence
* stock_confidence
* strength_weight
* market_weight
* (1 + sector_support)
```

含义分别是：

- `topic_confidence`：和该主题的相关程度
- `stock_confidence`：和该股票的相关程度
- `strength_weight`：`low=1, medium=2, high=3`
- `market_weight`：整体市场重要性，`low=1, medium=2, high=3`
- `sector_support`：如果该股票所在行业也被判断为受影响，就再加一层支持

所以：

- 相关性弱的新闻，进入图后的影响分数更低
- 影响强度高、市场相关性高的新闻，进入图后的影响分数更高
- 同一条新闻对不同股票和主题的连接强弱也会不同

## 10. 为什么 `mixed` 和 `uncertain` 记成 0

这是一个刻意保守的建模选择。

原因是如果一篇新闻结论本来就不明确，你不应该强行把它算成正面或负面。

这样做虽然保守，但更适合期末项目，因为：

- 容易解释
- 不容易误导
- 更符合“影响分析”而不是“预测”的定位

## 11. 这个层次在项目里有什么价值

它让项目从“图 + 新闻主题”进一步变成了：

- 图结构分析
- 新闻语义影响分析
- 可汇总、可展示的行业/股票影响结果

这一步对简历也有帮助，因为它说明你不只是做了传统图分析，还把现代 LLM 结构化抽取接进了系统。

## 10. 你应该能讲出来的话

> LLM 在这个项目里不是用来预测股价的，而是作为离线 enrichment 层，把新闻转成对行业和股票的结构化影响标签。它输出文章摘要、行业影响和股票影响三张表，再由查询网站把这些结果展示成行业和股票影响汇总，以及文章级明细。网站中的影响分数不是黑箱，而是由影响强度和置信度显式计算出来的。

如果老师继续追问“那如果来一条新的新闻怎么办”，你还应该能补一句：

> 我还额外做了一个受控的 New News Analysis 页面。用户输入一条新的新闻标题和摘要后，系统会让 LLM 在当前图的股票、行业和主题范围内做一次结构化影响评估，再把这些结果转换成合成 `topic_stock` 行，临时叠加到现有图结构上。因此它展示的是 graph-centered impact preview，不是价格预测。
